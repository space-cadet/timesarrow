## Details

Fully functional, self-contained numerics dashboard deployed to GitHub Pages. Replaced the Quarto+OJS prototype (T28) which had fragile runtime loading issues.

## Architecture
- Single static HTML file, zero build step, zero dependencies
- Vanilla JavaScript with DOM manipulation
- Data loaded via `fetch()` from external JSON files
- GitHub raw fallback for immediate availability (GitHub Pages has ~5min cache delay)
- Dark mode via `prefers-color-scheme` media query
- Mobile responsive design

## Features Implemented

### Runs & Results Tab
- Filterable runs table (Task, Phase, Status, Dimension, free-text search)
- Sortable columns (Run ID, Task, Phase, L, D, β Range, Wall, Perf, βs, Date, Status)
- Row selection with detail panel
- Detail panel: Parameters, Results, Key Findings, Timing, Export buttons
- JSON and CSV export for selected run
- Test run indicators ([TEST] prefix)

### Figure Archive Tab
- 6 FSS figure thumbnails with click-to-zoom modal
- Orthogonal filters: Task × Dimension × Type
- Search by title
- Figure count display

### Performance Tab
- SVG log-log scatter plot: Wall time vs Lattice Size
- Color-coded by dimension (2D=blue, 3D=green)
- Hover tooltips with run details

## Decisions
- Abandoned OJS/Quarto approach: cells rendered as raw code on static hosting, fragile runtime loading
- External JSON files instead of inline data: 235KB inline blob caused syntax errors and blank page
- Minimal inline fallback for offline use (not yet implemented — future enhancement)
- Vanilla JS over React: faster iteration, no build step, sufficient for current feature set
