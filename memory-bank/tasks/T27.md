## Description

## Implementation Details

**Location:** `timesarrow/rust-lattice/`

**Files:**
- `src/lib.rs` — Core library: Z2GaugeField, simulate_beta(), tests
- `src/main.rs` — CLI: parses args, runs parallel β sweep via rayon, outputs JSON
- `Cargo.toml` — Dependencies: rand 0.9, rand_xoshiro 0.7, serde, serde_json, rayon 1.10

**Key design decisions:**
- Flat Vec<i8> link storage (2 links per site: horizontal + vertical)
- Xoshiro256PlusPlus PRNG (fast, seedable, deterministic)
- Plaquette computed with periodic boundary conditions
- Each β value runs on a separate rayon thread; results collected in order
- Release profile: opt-level=3, LTO, codegen-units=1 for max performance

**Parallelization model:**
```
beta_values.par_iter().enumerate().map(|(i, beta)| {
    simulate_beta(l, *beta, thermal_sweeps, measure_sweeps, 10, 42 + i as u64)
}).collect()
```
Each thread gets its own Z2GaugeField instance → no mutex/contention.

## Performance
- L=8, 1k sweeps: ~2ms per β (wall time dominated by startup)
- **L=16, 100k sweeps, 11 β values: 3.0s total** (TypeScript: ~2h 10m)
- **Speedup: ~2,500–3,000× vs TypeScript worker threads**
- Binary size: 419 KB (release)

## Bug Fix History
1. **Plaquette geometry bug** (v1): Used wrong link indices for top/left edges (`link(x+1,y+1,0)` and `link(x,y+1,1)` instead of `link(x,y+1,0)` and `link(x,y,1)`)
2. **Sign convention fix** (v2): TypeScript uses 4 directed links per site; Rust uses 2 undirected links. Traversal conventions produce opposite raw signs. Fixed by negating `plaquette_stats()` output (NOT the plaquette function itself, which would break Metropolis dynamics)
3. **Test fix**: `test_flip_changes_plaquette` originally checked wrong plaquette indices
