## Objective

Add a new output mode that saves full link configurations (all σ_e = ±1), not just ensemble averages and time series. Required for flux loop analysis (T34b).

## Why New Data Is Needed

The existing T20/T31 JSON files contain ensemble averages and (where requested) plaquette time series, **not** link configurations. Flux loop analysis requires the raw σ_e values to compute plaquette fluxes B_p = ∏_{e∈∂p} σ_e and extract connected clusters.

## Implementation

### CLI Flags

```bash
--save-configs <directory>          # Enable snapshot saving
--snapshot-interval <n>             # Save every n sweeps
--snapshot-separation <m>           # Minimum separation between saved configs (in sweeps)
```

### Snapshot Format

Binary or compressed text. Each snapshot includes metadata:

```json
{
  "format_version": "1.0",
  "cell_complex_hash": "sha256:abc123...",
  "action_convention": "Wilson / Z2 standard: S = -β ∑_p ∏_e σ_e",
  "beta": 0.75,
  "L": 8,
  "lattice_type": "cubic",
  "thermalization_sweeps": 50000,
  "measurement_sweep_interval": 100,
  "autocorrelation_time": 42,
  "rng_seed": 12345,
  "sweep_number": 50200
}
```

### Sampling

Save **statistically separated configurations** (interval ≥ measured autocorrelation time), not fixed sweep intervals without checking.
