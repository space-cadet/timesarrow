## Objective

Build a general oriented cell-complex representation in `rust-lattice` that supports arbitrary valid 3D cell complexes, not merely hardcoded Cartesian lattices. The API must accept a valid oriented 3D cell complex with boundary operators over ℤ₂ as ground truth.

## Motivation

The current `rust-lattice` code assumes Cartesian plaquettes (`xy`/`yz`/`xz`). A 4-valent 3D complex like the diamond lattice has non-planar hexagonal plaquettes. Valence alone does not determine plaquettes or 3-cells — the cellulation must be specified and verified. This infrastructure is required before any CZX-linked numerics on 4-valent 3D graphs.

## Key Design Decisions

### Cell-Complex Types (Rust)

```rust
struct CellComplex {
    n_vertices: usize,
    n_edges: usize,
    n_plaquettes: usize,
    n_cells: usize,
    // Boundary operators over Z_2 as sparse CSR/CSC matrices
    d1: SparseMatrix,  // edges → vertices: n_v × n_e
    d2: SparseMatrix,  // plaquettes → edges: n_e × n_p
    d3: SparseMatrix,  // cells → plaquettes: n_p × n_c
}
```

### Ground-Truth Checks (Invariant Enforcement)

| Check | Mathematical Condition | Purpose |
|-------|----------------------|---------|
| Plaquette closure | `d1 · d2 = 0` | Every plaquette is a closed edge cycle |
| Bianchi identity | `d2 · d3 = 0` | Boundary of a 3-cell has zero boundary |
| Homology (periodic 3-torus) | `H_0 = 1`, `H_1 = 3`, `H_2 = 3`, `H_3 = 1` | Verify topology |

### Diamond Lattice Generator

- Generate the graph: two interpenetrating FCC sublattices (A and B), tetrahedral coordination
- **Derive plaquettes from the graph:** enumerate shortest closed cycles (hexagons)
- **Derive 3-cells from plaquettes:** find minimal closed volumes bounded by plaquettes
- **Do not assume** "tetrahedra + octahedra" — derive from the specified cellulation
- Verify Euler characteristic and homology ranks explicitly

## Notes

**Historical context:** This task emerged from a multi-agent discussion (Sage + GPT 5.6 Luna) about whether the CZX construction requires a 4-valent graph. Consensus: 4-valence is the minimal, cleanest case — not a mathematical necessity, but the natural setting for the CZX construction because the four-spin-1/2 singlet space is small (dimension 2). The diamond lattice is a natural 4-valent lattice in 3D with tetrahedral coordination.

**Key correction from review:** The original plan proposed "one lattice-constructor function" and assumed "dual is BCC." This was wrong — a general cell-complex API with boundary operators and homology verification is required.
