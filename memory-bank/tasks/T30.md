## Overview

Create a shared plotting infrastructure module (`numerics/src/plotting.py`) to eliminate code duplication across the 19 existing Python plotting/analysis scripts. This is Tier 1 of a three-tier unification plan.

## Motivation

Current state: 19 Python scripts with duplicated patterns:
- Hardcoded absolute paths in every script
- Repeated `load_data()` / `extract()` functions
- Inconsistent color definitions across scripts
- Varying figure sizes, DPIs, and save formats
- Boilerplate `tight_layout()` + `savefig()` in every script

Impact: Maintenance burden, inconsistent figures, brittle path references.

## Design Decisions

| Decision | Rationale |
|----------|-----------|
| Auto-detect project root | Eliminates hardcoded paths; works on any clone |
| Color palette per L value | Consistent visual identity across all figures |
| Colorblind-friendly hues | Accessibility; red-green safe |
| `load_run()` searches `output/` and `data/fss/` | Covers both coarse and fine scan data |
| `extract_observables()` returns numpy arrays | Ready for vectorized operations |
| Separate `save_figure()` and `save_publication()` | Quick iteration vs. final output |
| `generate_multi_panel()` for batch plots | Common pattern: 2×2 subplots of all observables |

## Notes

- This module is for **plotting infrastructure only**. Analysis scripts (curve fitting, exponent extraction, statistical tests) remain separate — they do physics, not visualization.
- Tier 2 (CLI unification) and Tier 3 (analysis pipeline) are out of scope for T30.
- The module is intentionally minimal. It provides primitives, not policy. Each script still controls what to plot and how.
