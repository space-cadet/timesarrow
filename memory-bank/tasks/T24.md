## Description
Simulate domain walls between regions of opposite Z₂ time orientation and measure their properties. The paper conjectures that these walls host "all-fermion toric code" surface states — this simulation tests whether the surface physics matches the SPT prediction.

## Acceptance Criteria
- [ ] Create initial state with a planar domain wall separating +σ_e and -σ_e regions
- [ ] Measure domain wall tension σ_DW as function of K - K_c
- [ ] Verify σ_DW ∝ (K - K_c) near criticality (mean-field exponent)
- [ ] Compute anyonic braiding statistics on the domain wall surface
- [ ] Check for fermionic self-statistics (exchange phase = -1 for quasi-particles)
- [ ] Compare with all-fermion toric code predictions
- [ ] Document whether the surface order is gapped (as predicted for bosonic SPT)

## Implementation Details
**Domain wall setup:**
- 3D lattice with fixed boundary conditions: σ_e = +1 for z < L_z/2, σ_e = -1 for z > L_z/2
- Equilibrate at fixed K > K_c
- Measure excess energy per unit area = domain wall tension

**Surface diagnostics:**
1. **Tension:** σ_DW = (E_DW - E_bulk) / Area
2. **Surface Wilson loops:** W_surface(γ) = ⟨∏_{e∈γ} σ_e⟩ for loops on the domain wall
3. **Anyon creation operators:** String operators creating e, m, and ε particles on the wall
4. **Braiding phases:** Compute mutual statistics via interferometric (Aharonov-Bohm) phases

**Expected results:**
- Tension scaling: σ_DW ~ (K - K_c)^μ with μ ≈ 1 (mean-field)
- Surface topological order: All three non-trivial anyons have fermionic statistics
- Gapped spectrum: No gapless modes (consistent with bosonic SPT)

**Comparison with theory:**
- Standard toric code: e and m are bosons, ε = e×m is fermion
- All-fermion toric code (predicted): e, m, and ε are all fermions
- The paper's conjecture predicts the latter.
